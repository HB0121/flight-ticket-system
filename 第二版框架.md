# 第二版框架

> 选题编号：3.1.7  
> 题目：基于网络爬虫的机票抓取与自动更新系统设计与实现  
> 阶段目标：在第一版最小闭环基础上，完成真实航班报价采集、DeepSeek AI 助手和 PC 端增强演示。

## 1. 第二版目标

第二版继续保持本地稳定演示，不引入公网部署和微信小程序，重点补强课堂答辩最容易被关注的三条链路：

1. 采集链路从本地样例 HTML 扩展为 Amadeus Flight Offers Search 真实航班报价 API。
2. 数据层新增价格快照，支持同一航线多次采集后的价格趋势展示。
3. AI 助手接入 DeepSeek API，未配置密钥或接口失败时回退到本地规则。
4. Vue 3 PC 端升级为采集配置、航班列表、航班详情、价格趋势、AI 出行建议和购票时机报告一体化界面。

第二版不做 Redis、完整向量库、微信小程序和公网服务器部署。这些内容保留到第三阶段或最终扩展，避免一次迭代范围过大。

## 2. 技术栈

| 模块 | 技术 | 说明 |
|---|---|---|
| 数据库 | MySQL 8 | 保存航班当前快照、价格历史快照和采集任务 |
| 真实采集 | Python 3.11 + Requests + Amadeus API | 通过官方 API 获取航班报价，保留 Scrapy 样例兜底 |
| 后端 | Java 17 + SpringBoot 3 + JDBC | 提供采集触发、航班查询、价格历史和 AI 接口 |
| AI | DeepSeek Chat Completions API | 生成出行建议和购票时机分析，失败时本地规则兜底 |
| 前端 | Vue 3 + Element Plus + ECharts | PC 端本地演示界面 |
| 发布 | GitHub + GitHub Pages | 上传代码和阶段文档 |

## 4. 数据模型

### 4.1 flight

`flight` 继续保存航班当前状态，字段沿用第一版：

| 字段 | 含义 |
|---|---|
| id | 主键 |
| flight_no | 航班号 |
| airline_name | 航司名称 |
| from_city | 出发城市 |
| to_city | 到达城市 |
| from_airport | 出发机场 |
| to_airport | 到达机场 |
| depart_time | 起飞时间 |
| arrive_time | 到达时间 |
| price | 当前价格 |
| seats_left | 剩余座位 |
| data_source | 数据来源，第二版支持 `sample` 和 `amadeus` |
| collected_at | 最近采集时间 |

唯一约束仍为 `flight_no + depart_time + data_source`。重复采集时更新当前价格和余票。

### 4.2 flight_price_snapshot

新增价格快照表，用于展示价格趋势和支撑购票时机分析。

| 字段 | 含义 |
|---|---|
| id | 主键 |
| flight_id | 关联 `flight.id` |
| flight_no | 航班号 |
| from_city | 出发城市 |
| to_city | 到达城市 |
| depart_time | 起飞时间 |
| price | 本次观测价格 |
| seats_left | 本次观测余票 |
| data_source | 数据来源 |
| observed_at | 观测时间 |

### 4.3 crawl_job

`crawl_job` 保留第一版状态字段，并扩展采集来源和请求参数：

| 字段 | 含义 |
|---|---|
| source | `sample` 或 `amadeus` |
| request_params | 本次采集的出发地、目的地、日期等参数摘要 |

## 5. REST API

| 方法 | 路径 | 用途 |
|---|---|---|
| POST | `/api/crawl/run` | 触发采集，支持 `source/fromCity/toCity/date/adults/maxResults` |
| GET | `/api/crawl/latest` | 查看最近一次采集任务 |
| GET | `/api/flights?fromCity=&toCity=&date=&dataSource=` | 查询航班列表 |
| GET | `/api/flights/{id}` | 查询航班详情 |
| GET | `/api/flights/{id}/price-history` | 查询单个航班价格历史 |
| POST | `/api/ai/advice` | 输入自然语言，返回推荐航班和出行建议 |
| POST | `/api/ai/timing` | 输入出行需求，返回购票时机分析报告 |

## 6. 外部配置

第二版新增以下环境变量：

| 变量 | 用途 | 必填 |
|---|---|---|
| `AMADEUS_CLIENT_ID` | Amadeus API Key | 否，缺失时使用 sample 兜底 |
| `AMADEUS_CLIENT_SECRET` | Amadeus API Secret | 否，缺失时使用 sample 兜底 |
| `AMADEUS_BASE_URL` | Amadeus API 地址，默认测试环境 | 否 |
| `DEEPSEEK_API_KEY` | DeepSeek API Key | 否，缺失时使用本地规则 |
| `DEEPSEEK_BASE_URL` | DeepSeek API 地址，默认 `https://api.deepseek.com` | 否 |
| `DEEPSEEK_MODEL` | DeepSeek 模型，默认 `deepseek-v4-flash` | 否 |

## 7. 演示流程

1. 启动 MySQL：`docker compose -f infra/docker-compose.yml up -d mysql`
2. 配置可选环境变量：Amadeus 和 DeepSeek API Key。
3. 启动后端：`cd backend && mvn spring-boot:run`
4. 启动前端：`cd frontend && npm.cmd install && npm.cmd run dev`
5. 打开 `http://localhost:5173`。
6. 在采集配置中选择 Amadeus，输入“上海 → 北京”和未来日期，点击采集。
7. 查看航班列表、航班详情和价格趋势图。
8. 输入出行需求，查看 DeepSeek 或本地规则生成的出行建议。
9. 打开购票时机分析，查看价格趋势摘要、风险等级和购买窗口建议。
