# 第一版框架

> 选题编号：3.1.7  
> 题目：基于网络爬虫的机票抓取与自动更新系统设计与实现  
> 阶段目标：先完成一个稳定、可课堂演示的最小闭环。

## 1. 第一版目标

第一版只做课堂演示所需的核心链路：

1. Scrapy 模拟爬虫解析本地样例 HTML，采集机票数据。
2. 爬虫将航班数据写入 MySQL。
3. SpringBoot 提供航班查询、爬虫触发、采集状态、AI 出行建议接口。
4. Vue 3 PC 端展示采集状态、航班列表、价格图表和 AI 建议结果。

第一版不做小程序、Redis、真实网页爬取、RAG 购票时机预测和公网服务器部署，避免演示受反爬、网络波动或额外环境影响。

## 2. 技术栈

| 模块 | 技术 | 说明 |
|---|---|---|
| 数据库 | MySQL 8 | Docker 启动，保存航班和采集任务 |
| 爬虫 | Python 3.11 + Scrapy | Docker 运行，解析本地样例 HTML |
| 后端 | Java 17 + SpringBoot 3 | 本机运行，提供 REST API |
| 前端 | Vue 3 + Element Plus + ECharts | 本机运行，提供 PC 演示界面 |
| 发布 | GitHub + GitHub Pages | 公开展示项目文档和代码 |

## 4. 数据模型

### 4.1 flight

保存航班数据。第一版字段包括：

| 字段 | 含义 |
|---|---|
| id | 主键 |
| flight_no | 航班号 |
| airline_name | 航司名称 |
| from_city | 出发城市 |
| to_city | 到达城市 |
| from_airport | 出发机场 |
| to_airport | 到达机场 |
| depart_time | 起飞时间 |
| arrive_time | 到达时间 |
| price | 价格 |
| seats_left | 剩余座位 |
| data_source | 数据来源 |
| collected_at | 采集时间 |

唯一约束：`flight_no + depart_time + data_source`。重复采集时更新价格、余票和采集时间。

### 4.2 crawl_job

保存采集任务状态。第一版字段包括：

| 字段 | 含义 |
|---|---|
| id | 主键 |
| status | RUNNING / SUCCESS / FAILED |
| started_at | 开始时间 |
| finished_at | 结束时间 |
| success_count | 成功采集数量 |
| failed_count | 失败数量 |
| error_message | 错误信息 |

## 5. REST API

| 方法 | 路径 | 用途 |
|---|---|---|
| POST | `/api/crawl/run` | 触发样例爬虫采集 |
| GET | `/api/crawl/latest` | 查看最近一次采集任务 |
| GET | `/api/flights?fromCity=&toCity=&date=` | 查询航班列表 |
| GET | `/api/flights/{id}` | 查询航班详情 |
| POST | `/api/ai/advice` | 输入自然语言，返回推荐航班和出行建议 |

AI 建议第一版采用本地规则：从自然语言中提取出发城市、到达城市、日期和预算，再查询数据库，优先推荐预算内价格最低的航班。后续可替换为讯飞星火或 Kimi API。

## 6. 演示流程

1. 启动 MySQL：`docker compose -f infra/docker-compose.yml up -d mysql`
2. 启动后端：`cd backend && mvn spring-boot:run`
3. 启动前端：`cd frontend && npm.cmd install && npm.cmd run dev`
4. 打开前端页面，点击“采集”。
5. 页面展示最近采集状态和航班列表。
6. 输入“2026-06-19 上海到北京，预算1200元”，展示 AI 推荐航班。
