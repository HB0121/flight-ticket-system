## 综合课程设计 III

## 1 目的

通过该课程设计，达到如下目的：

（1）学会通过网络、 书籍等渠道和途径进行知识获取与应用的方法；

（2）灵活运用专业知识与技术解决计算机应用实际问题的能力。

（3）通过培养学生创新能力、实践能力、工程应用能力，从而提高学生的专业综合能力与水平；

（4）理解和掌握至少一种常用 Web 应用程序开发框架（如SMM）和开发流程；

（5）理解和掌握 Web 应用程序中前端和服务端的通信方式；

（6）理解和掌握关系型数据库在 Web 应用系统中的开发方法；

（7）培养开发 Web 应用系统所需的工程能力， 以及规范化和模块化意识；

（8）培养团队沟通与合作能力、 自我学习能力以及独立解决问题的能力。

（9）培养学生利用大模型辅助编程的能力。

## 2.2 选题规则

（1）选题原则： 每个学生完成的题目为学号最后两位数字%A区题目总数+1。例如，如果 A区题目总数为 16 题，对于学号为 12303070101 的同学所选择的题目号则为：01%15+1=2，则该生的选题为 A 区第 2 题。

（2）分组原则：题号相同的同学可以为同一个组，出于工作量的考虑，一个组最多两个人，组内同学的工作分工由同学自己决定，但是在课程设计报告中必须明确写清楚。

（3）题目选定后原则上不允许随意更换题目，对于个别需要更换题目的同学，可以在跟指导商量并征的指导老师同意后方可更换。

## 2.3 关于团队合作

（1）最多两个人一个组，也可以独立完成，对于独立完成的同学，将不能获得团队合作部分的成绩（占总成绩的 15%）

（2）对于前端功能过于简单的系统， 分工不不建议按照前端和后台来分工，只能按照功能来分工。分工原则为按照系统总体工作量的多少，每个人完成 50%左右的工作为宜。

（3） 同一组同学的代码托管在 github（或 gitee）上，课程文档中需要把每个人的代码提交记录截图放进去， 以展示每个同学的工作量。

## 3.1.7 基于网络爬虫的机票抓取与自动更新系统设计与实现

本系统主要满足以下功能性需求：

## ① 抓取可靠的机票数据

这是本系统最核心、最重要的工作。这需要提供收集数据的可靠方法和验证措施，并且对收集到的数据进行预处理，观察其是否符合实际情况，最后对异常数据进行清洗和删除，保证数据的可靠性和真实性。

③ 提供机票数据接口

API 是软件到软件的接口，而不是用户界面。使用 API ，应用程序在没有任何用户知识或干预的情况下相互通信。在借助 API 的情况下，用户可以通过网络轻松获得该系统之前抓取的机票数据，进行下一步的处理。

## ④ 前端页面展示

在获得了机票数据接口之后，我们需要针对网上店铺不同的筛选查询条件，为其提供合适的接口，方便网上店铺后台可以及时将机票价格及航班信息逐条更新到网上店铺前台中。
